# 小派 AI 记忆策略

## 概述
小派（PillPal AI 助手）使用三层上下文管理策略，在保持对话连贯性的同时控制 token 成本。

## 三层记忆架构

### 第1层：用户画像（永久层）
**内容**：用户基本信息 + 药品信息，不随对话变化
**更新时机**：登录时加载，添加/删除药品时刷新
**约 100 tokens**

```
【用户画像】
姓名：张弓长，58岁，男
疾病：高血压、2型糖尿病
药品：
- 氨氯地平片 5mg，空腹，治高血压（晨起/早餐后/午餐后/晚间）
- 氯雷他定 10mg，无要求，治过敏（晚间）
用药开始：2026-05-01
```

**数据来源**：
- profiles 表 → 姓名
- user_metadata → 年龄、性别、疾病
- medications 表 → 药品列表

---

### 第2层：实时数据（每次刷新）
**内容**：今日打卡状态 + 库存预警，每次提问时从 store 取最新数据
**约 100 tokens**

```
【今日状态】5次药中已服2次
已服：氨氯地平(7:00) 11:30、氨氯地平(8:00) 11:30
待服：氨氯地平(14:30)、氨氯地平(21:00)、氯雷他定(21:00)
【库存】氨氯地平 26片/6天⚠、氯雷他定 28片/28天✅
```

**数据来源**：
- medications store → 库存
- records store → 今日打卡记录

---

### 第3层：对话历史（滑动窗口 + 压缩）
**内容**：历史对话，超过 token 阈值时自动压缩旧轮次为摘要

#### 压缩策略

**触发条件**：历史对话总 token > 4000（约 15-20 轮）

**压缩流程**：
1. 计算历史对话总 token（中文：字数 × 1.5）
2. 超过 4000 tokens → 触发压缩
3. 保留最近 6 轮完整对话（一问一答 = 一轮）
4. 把第 6 轮之前的所有旧轮次打包，发给 DeepSeek：
   - prompt："请用一段话总结以下对话的要点，包括用户问了什么、做了什么操作、得到了什么建议"
5. DeepSeek 返回摘要（约 50-100 tokens）
6. 新的历史 = [摘要消息] + [最近 6 轮完整对话]

**压缩前（20轮，6000 tokens）**：
```
messages = [
  { system: "..." },
  { user: "库存怎么样？" },
  { assistant: "氨氯地平剩26片..." },
  { user: "帮我打卡晨起的" },
  { assistant: "已打卡..." },
  ... 还有16轮 ...
  { user: "当前问题" }
]
```

**压缩后（摘要 + 6轮，约 1500 tokens）**：
```
messages = [
  { system: "..." },
  { assistant: "[对话摘要] 用户查过库存（氨氯地平26片），打过晨起和早餐后的卡，问过能否和酒同服（建议不要），修改过氯雷他定库存为30片。" },
  // 最近6轮完整对话
  { user: "..." },
  { assistant: "..." },
  ... 共12条 ...
  { user: "当前问题" }
]
```

**多次压缩**：如果继续聊，又超过 4000 tokens，再次压缩。新摘要会包含旧摘要的内容（因为旧摘要在被压缩的对话里）。

## Token 预算

| 层 | 约 tokens | 说明 |
|---|---|---|
| System prompt（角色+工具定义） | ~300 | 固定 |
| 第1层：用户画像 | ~100 | 登录时生成 |
| 第2层：实时数据 | ~100 | 每次刷新 |
| 第3层：历史摘要 | ~100 | 压缩后 |
| 第3层：最近6轮对话 | ~600 | 保留完整 |
| 当前用户输入 | ~50 | 当次问题 |
| **总计** | **~1250** | 稳定，不会无限增长 |

---

## API 调用成本

| 场景 | API 调用次数 | 成本 |
|---|---|---|
| 普通问答 | 1次 | ~0.001 元 |
| 操作（打卡等） | 2次（判断+回复） | ~0.002 元 |
| 触发压缩 | 额外1次 | ~0.001 元 |
| 每天聊20轮 | 约25次 | ~0.025 元 |

DeepSeek 价格：1元/百万输入 token，2元/百万输出 token
