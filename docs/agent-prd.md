# 小派 Agent PRD（产品需求文档）

## 1. 产品定位

**从 Chatbot 升级为 Agent**：小派不再只是"你问我答"，而是一个能**主动思考、自主规划、连续执行**的用药管理智能体。

### Chatbot vs Agent

```
Chatbot（现在）：
  用户 → "帮我打卡" → AI 调1个函数 → 回复 → 结束

Agent（目标）：
  用户 → "帮我看看今天的情况"
       → AI 思考：需要查什么？
       → Step 1: 查打卡状态 → 发现3个没吃
       → Step 2: 查库存 → 发现氨氯地平快没了
       → Step 3: 查上次服药时间 → 发现昨晚漏服
       → 综合分析 → 生成建议
       → 等用户确认 → 执行操作
```

## 2. Agent 能力清单

### 2.1 感知能力（获取信息）

| 工具 | 说明 | 触发场景 |
|---|---|---|
| `get_medications` | 查用户的药品列表 | "我吃什么药" |
| `get_today_status` | 查今日打卡状态 | "今天吃了几次" |
| `get_stock` | 查库存和预警 | "药还够吃吗" |
| `get_history` | 查某天的服药记录 | "昨天吃药了吗" |
| `get_user_profile` | 查用户基本信息 | "我的疾病是什么" |

### 2.2 执行能力（操作数据）

| 工具 | 说明 | 触发场景 |
|---|---|---|
| `take_med` | 打卡服药 | "吃了" |
| `skip_med` | 跳过服药 | "今天不吃了" |
| `undo_med` | 撤回打卡 | "打错了" |
| `update_stock` | 修改库存 | "买了30片" |
| `add_medication` | 添加新药品 | "医生又开了一种新药" |
| `remove_medication` | 删除药品 | "这个药停了" |

### 2.3 分析能力（推理判断）

| 能力 | 说明 | 示例 |
|---|---|---|
| 服药分析 | 分析服药规律和依从性 | "我这周吃药情况怎么样" |
| 库存预测 | 预测何时断药 | "药什么时候吃完" |
| 风险评估 | 综合评估用药风险 | "帮我看看有没有问题" |
| 用药建议 | 基于数据给出建议 | "我需要注意什么" |

### 2.4 主动能力（未来）

| 能力 | 说明 | 实现方式 |
|---|---|---|
| 定时提醒 | 到点主动提醒吃药 | 推送通知 |
| 异常检测 | 发现漏服主动关心 | 后台检查 |
| 续方提醒 | 库存不足主动建议 | 库存监控 |

## 3. Agent 思维链（ReAct 模式）

### 核心循环

```
用户输入
  → Thought（思考）：分析用户意图，决定需要做什么
  → Action（行动）：调用一个或多个工具
  → Observation（观察）：看到工具返回的结果
  → Thought（再思考）：需要继续做什么？还是可以回复了？
  → ... 循环直到任务完成 ...
  → Response（回复）：给用户最终回复
```

### 示例场景

**场景1：简单问答**
```
用户："我吃什么药？"
Thought: 用户想知道药品列表，调 get_medications
Action: get_medications()
Observation: [氨氯地平片 5mg, 氯雷他定 10mg]
Thought: 信息够了，可以回复
Response: "你目前在吃2种药：氨氯地平片 5mg（治高血压）和氯雷他定 10mg ..."
```

**场景2：多步操作**
```
用户："帮我看看今天的情况"
Thought: 需要查今日状态和库存
Action: get_today_status()
Observation: 5次中已服2次，待服3次
Thought: 还需要查库存情况
Action: get_stock()
Observation: 氨氯地平剩6天⚠，氯雷他定剩28天✅
Thought: 有库存预警，需要综合分析给建议
Response: "今天还有3次药没吃哦... 另外氨氯地平只剩6天了，建议尽快续方 ⚠"
```

**场景3：复杂任务**
```
用户："医生让我加一种新药，降糖药二甲双胍，每天3次饭后吃"
Thought: 用户要添加新药，需要调 add_medication
Action: add_medication(name="二甲双胍", dosage="0.5g", times=["早餐后","午餐后","晚餐后"], condition="餐后30分钟", disease="糖尿病", stock=0)
Observation: 添加成功
Thought: 库存是0，需要提醒用户买药
Action: get_stock()
Observation: 二甲双胍 0片/0天
Thought: 应该提醒用户购药
Response: "已帮你添加二甲双胍到用药计划 ✅ 每天3次饭后吃。不过库存是0哦，记得先去买药 🏥"
```

**场景4：带确认的操作**
```
用户："把氨氯地平停了"
Thought: 停药是重要操作，需要确认
Response: "停用氨氯地平是医生建议的吗？⚠️ 高血压药物不建议自行停药。如果是医生建议，我帮你从计划中移除。请确认：是/否"
用户："是的，医生说换药了"
Thought: 用户确认了，执行删除
Action: remove_medication(name="氨氯地平")
Observation: 删除成功
Response: "已移除氨氯地平 ✅ 记得按医生建议开始新的用药方案哦"
```

## 4. 技术实现方案

### 4.1 循环调用架构

```
┌─────────────────────────────────────┐
│           用户输入                    │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│    DeepSeek API (带 tools 定义)      │
│    System: 用户画像 + 实时数据        │
│    History: 压缩后的对话历史          │
│    User: 当前问题                    │
└──────────────┬──────────────────────┘
               ▼
         ┌─────────────┐
         │ AI 返回什么？ │
         └──────┬──────┘
                │
        ┌───────┼───────┐
        ▼               ▼
  text 回复        tool_calls
  (直接回复)      (调用工具)
        │               │
        ▼               ▼
   返回给用户      执行工具，拿到结果
                        │
                        ▼
                  把结果发回给 AI
                  (tool role 消息)
                        │
                        ▼
                  AI 再次判断
                  ├── 还要调工具 → 循环
                  └── 可以回复了 → 返回给用户
```

### 4.2 与现在的区别

| | 现在（单次调用） | Agent（循环调用） |
|---|---|---|
| API 调用次数 | 1-2次 | 1-N次（直到任务完成） |
| 工具调用 | 最多调1个 | 可以连续调多个 |
| 上下文 | 不带工具返回结果 | 每次工具结果都追加到消息里 |
| 思考过程 | 用户看不到 | 可以展示"正在查询库存..." |

### 4.3 最大循环次数

设置 **maxSteps = 5**，防止 AI 无限循环。超过5步强制返回。

## 5. 实施计划

### Phase 1：Agent 循环引擎（核心）
- 改造 `askAI` 为循环调用，支持多步工具调用
- 工具执行结果以 `tool` role 消息追加
- 前端展示思考步骤（"正在查询..."）
- 最大5步限制

### Phase 2：扩展工具集
- 添加 `get_medications`、`get_today_status`、`get_stock`、`get_history` 感知工具
- 添加 `add_medication`、`remove_medication` 操作工具
- AI 可以自由组合这些工具完成复杂任务

### Phase 3：智能确认
- 危险操作（删药、停药）要求 AI 先确认再执行
- AI 在 system prompt 里被告知哪些操作需要确认

### Phase 4：主动能力（未来）
- 定时检查打卡状态
- 库存预测和主动提醒
- 个性化用药建议

## 6. 成功指标

| 指标 | 目标 |
|---|---|
| 单次对话平均工具调用次数 | 1.5-3次（说明 AI 会组合工具） |

| 平均步数 | 2-3步完成大多数任务 |
